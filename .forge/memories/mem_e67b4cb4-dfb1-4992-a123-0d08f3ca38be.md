The `excalidraw-assistant` tool architecture is finalised as validate-and-render only: the agent (LLM) generates Excalidraw JSON by reading reference files, writes it to disk, then calls the tool with `--input <path>`. The Go tool validates the JSON and renders it to PNG via the Python/Playwright pipeline. No color palette validation is done tool-side — the agent is trusted to read and follow `color-palette.md`, and users may explicitly override colors. This was a deliberate user decision: 'I dont think we need to add validation against color palette, that would be complex. lets trust the agent to read the color palette and follow, and let it use different when the user tells it to do different'.
